1. 새로운 .gitignore 파일 생성:
  # 프로젝트 루트 디렉터리에 .gitignore 파일을 생성합니다.

2. 파일 또는 디렉터리 추가:
  # .gitignore 파일에 Git에서 추적하지 않을 파일이나 디렉터리를 추가합니다. 각 줄에 하나의 항목을 추가합니다.
  ```
  node_modules/
  .env
  build/
  ```

  # 위 예시에서는 node_modules/, .env, build/ 디렉터리와 파일들을 Git에서 추적하지 않도록 설정합니다.

3. 중요한 점:
  .gitignore 파일은 주석으로 #을 사용할 수 있습니다.
  * 와일드카드는 여러 파일을 나타냅니다. 예를 들어, *.log는 확장자가 .log인 모든 파일을 나타냅니다.
  !를 사용하여 특정 파일이나 디렉터리를 무시하지 않도록 설정할 수 있습니다. 예를 들어, !important-file.txt는 important-file.txt 파일을 무시하지 않도록 설정합니다.

4. 적용:
  # .gitignore 파일을 저장한 후, Git이 해당 파일들을 추적하지 않도록 설정됩니다.

5. 이미 Git에 추가된 파일에는 적용되지 않음:
  # 이미 Git에 추가된 파일에 대해서는 .gitignore를 적용해도 변화가 없습니다. 이는 이미 추적되고 있는 파일들을 다시 추적하지 않도록 하기 위함입니다.

# .gitignore 파일을 사용하면 프로젝트에서 Git으로 관리하지 않아도 되는 임시 파일이나 빌드 결과물, 개인 설정 등을 무시하고, 깔끔한 프로젝트 구조를 유지할 수 있습니다.
